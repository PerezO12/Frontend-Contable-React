# Servicio de Exportaci√≥n - Documentaci√≥n T√©cnica

## Descripci√≥n General

El `ExportService` es el componente central del sistema de exportaci√≥n que proporciona capacidades robustas y seguras para exportar datos de cualquier tabla del sistema contable en m√∫ltiples formatos, con filtrado autom√°tico de informaci√≥n sensible.

## Caracter√≠sticas Principales

### üîÑ Exportaci√≥n Gen√©rica
- **Soporte universal**: Compatible con todas las tablas del sistema
- **M√∫ltiples formatos**: CSV, JSON, XLSX
- **Filtrado flexible**: Por IDs, fechas, estado y filtros personalizados
- **Metadatos completos**: Informaci√≥n detallada de cada exportaci√≥n

### üîí Seguridad Integrada
- **Filtrado autom√°tico**: Omisi√≥n de campos sensibles como contrase√±as y tokens
- **Control granular**: Configuraci√≥n espec√≠fica por tabla
- **Validaci√≥n de datos**: Verificaci√≥n de integridad antes de exportar

### ‚ö° Rendimiento Optimizado
- **Consultas eficientes**: Uso de SQLAlchemy para optimizaci√≥n
- **Procesamiento en memoria**: Generaci√≥n de archivos sin almacenamiento temporal
- **Paginaci√≥n**: Soporte para grandes vol√∫menes de datos

## Arquitectura del Servicio

```python
class ExportService:
    """Servicio para exportaci√≥n gen√©rica de datos de la base de datos"""
    
    # Configuraci√≥n est√°tica

    _build_query()          # Construcci√≥n de consultas
    _model_to_dict()        # Conversi√≥n modelo ‚Üí diccionario
```

## M√©todos Principales

### `get_available_tables() -> AvailableTablesResponse`

Obtiene informaci√≥n de todas las tablas disponibles para exportaci√≥n.

**Funcionalidad:**
- Lista todas las tablas configuradas
- Obtiene metadatos de cada tabla (columnas, registros totales)
- Filtra campos sensibles en la informaci√≥n de esquema
- Maneja errores graciosamente

**Retorna:**
```python
AvailableTablesResponse(
    tables=[
        TableSchema(
            table_name="users",
            display_name="Usuarios",
            description="Tabla Usuarios",
            available_columns=[...],
            total_records=150
        )
    ],
    total_tables=10
)
```

### `get_table_schema(table_name: TableName) -> TableSchema`

Obtiene el esquema detallado de una tabla espec√≠fica.

**Par√°metros:**
- `table_name`: Nombre de la tabla (enum TableName)

**Funcionalidad:**
- Inspecciona la estructura de la tabla
- Filtra campos sensibles
- Incluye datos de muestra (5 registros)
- Proporciona metadatos completos

**Retorna:**
```python
TableSchema(
    table_name="users",
    display_name="Usuarios",
    description="Tabla Usuarios",
    available_columns=[
        ColumnInfo(name="id", data_type="number", include=True),
        ColumnInfo(name="email", data_type="string", include=True),
        # hashed_password omitido por seguridad
    ],
    total_records=150,
    sample_data=[{...}, {...}]  # Sin campos sensibles
)
```

### `export_data(request: ExportRequest, user_id: UUID) -> ExportResponse`

M√©todo principal para exportar datos seg√∫n par√°metros especificados.

**Par√°metros:**
- `request`: Especificaci√≥n de la exportaci√≥n (tabla, formato, filtros)
- `user_id`: ID del usuario que realiza la exportaci√≥n

**Proceso:**
1. **Validaci√≥n**: Verifica par√°metros de entrada
2. **Consulta**: Construye query con filtros aplicados
3. **Obtenci√≥n**: Ejecuta consulta y obtiene datos
4. **Transformaci√≥n**: Convierte modelos a diccionarios con filtrado de seguridad
5. **Generaci√≥n**: Crea archivo en formato especificado
6. **Metadatos**: Genera informaci√≥n de la exportaci√≥n
7. **Respuesta**: Retorna archivo y metadatos

**Ejemplo de uso:**
```python
request = ExportRequest(
    table_name=TableName.USERS,
    export_format=ExportFormat.CSV,
    filters=ExportFilter(
        ids=[1, 2, 3],
        active_only=True
    )
)

response = service.export_data(request, user_id)
```

**Retorna:**
```python
ExportResponse(
    file_name="users_20241210_143022.csv",
    file_content="id,email,first_name,last_name\n...",

)
```

## M√©todos de Filtrado y Seguridad

### `_filter_sensitive_columns(columns, table_name) -> List[ColumnInfo]`

Filtra autom√°ticamente columnas sensibles seg√∫n la configuraci√≥n de la tabla.

**Funcionalidad:**
- Consulta configuraci√≥n SENSITIVE_FIELDS para la tabla
- Excluye campos sensibles de la lista de columnas
- Registra campos omitidos para auditor√≠a
- Retorna lista filtrada de columnas seguras

### `_model_to_dict(model_instance, selected_columns, table_name) -> Dict`

Convierte instancia de modelo SQLAlchemy a diccionario con filtrado de seguridad.

**Funcionalidad:**
- Maneja conversi√≥n de tipos especiales (Decimal, DateTime, UUID)
- Aplica filtrado de campos sensibles autom√°ticamente
- Gestiona relaciones y campos nulos
- Optimiza serializaci√≥n para exportaci√≥n

## M√©todos de Generaci√≥n de Archivos

### `_generate_json(data) -> str`
Genera archivo JSON con codificaci√≥n UTF-8 y formato legible.

### `_generate_csv(data) -> str`
Genera archivo CSV con encabezados y separadores est√°ndar.

### `_generate_xlsx(data) -> bytes`
Genera archivo Excel usando pandas y openpyxl con formato profesional.

## Construcci√≥n de Consultas

### `_build_query(model, filters) -> Query`

Construye consultas SQLAlchemy optimizadas con filtros aplicados.

**Filtros soportados:**
- **IDs espec√≠ficos**: `WHERE id IN (1,2,3)`
- **Rango de fechas**: `WHERE created_at BETWEEN date1 AND date2`
- **Estado activo**: `WHERE is_active = true/false`
- **Filtros personalizados**: Campos espec√≠ficos con valores
- **Paginaci√≥n**: OFFSET y LIMIT para grandes datasets

**Optimizaciones:**
- Uso de √≠ndices para filtros de ID
- Filtros de fecha optimizados
- Validaci√≥n de existencia de campos antes de aplicar filtros

## Manejo de Errores

El servicio implementa manejo robusto de errores:

```python
try:
    # Operaci√≥n de exportaci√≥n
    result = export_operation()
except SQLAlchemyError as e:
    # Error de base de datos
    raise ExportError(f"Error de base de datos: {str(e)}")
except ValidationError as e:
    # Error de validaci√≥n
    raise ExportError(f"Datos inv√°lidos: {str(e)}")
except Exception as e:
    # Error general
    raise ExportError(f"Error en la exportaci√≥n: {str(e)}")
```

## Consideraciones de Rendimiento

### Optimizaciones Implementadas
- **Consultas lazy**: Solo carga datos necesarios
- **Filtros a nivel de base de datos**: Reduce transferencia de datos
- **Generaci√≥n en memoria**: Evita I/O de archivos temporales
- **Conversi√≥n eficiente**: Minimiza transformaciones de datos

### L√≠mites Recomendados
- **CSV/JSON**: Hasta 100,000 registros
- **XLSX**: Hasta 50,000 registros (limitaci√≥n de Excel)
- **Memoria**: Monitoreo de uso para datasets grandes

## Seguridad y Auditor√≠a

### Medidas de Seguridad
- **Filtrado autom√°tico**: Campos sensibles nunca se exportan
- **Validaci√≥n de entrada**: Todos los par√°metros son validados
- **Control de acceso**: Integraci√≥n con sistema de autenticaci√≥n
- **Logs de auditor√≠a**: Registro de todas las exportaciones

### Campos Protegidos
- Contrase√±as y hashes
- Tokens de sesi√≥n y API
- Claves privadas y secretos
- Informaci√≥n de configuraci√≥n sensible
